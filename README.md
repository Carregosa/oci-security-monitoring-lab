# Oracle Cloud Infrastructure Security Monitoring Lab

## Overview

This project demonstrates practical usage of Oracle Cloud Infrastructure as part of my Oracle ACE Apprentice journey.

The lab includes the creation of a basic OCI environment using networking, compute, object storage, monitoring and logging resources.

## Objective

The objective of this lab is to demonstrate hands-on experience with Oracle Cloud Infrastructure and document a simple, secure and repeatable learning path for beginners.

## Oracle Cloud Services Used

- Oracle Cloud Infrastructure
- OCI Compute
- OCI Virtual Cloud Network
- OCI Object Storage
- OCI Monitoring
- OCI Logging

## Lab Architecture

The environment includes:

- Dedicated OCI compartment
- Virtual Cloud Network
- Public subnet
- Internet gateway
- Security rules
- Compute instance
- Object Storage bucket
- Monitoring metrics
- Logging resources

## Steps Performed

1. Created a dedicated OCI compartment.
2. Created a Virtual Cloud Network.
3. Configured a public subnet and internet gateway.
4. Reviewed security rules.
5. Created a Compute instance using Oracle Linux.
6. Created an Object Storage bucket.
7. Uploaded a test file to the bucket.
8. Reviewed compute monitoring metrics.
9. Reviewed logging and observability options.
10. Captured screenshots as evidence of product usage.

## Security Considerations

This lab follows basic security principles:

- Use of a dedicated compartment.
- Controlled network access.
- Review of ingress rules.
- Separation of resources.
- Monitoring and logging visibility.
- No sensitive data exposed in screenshots.

This lab demonstrates practical product usage of Oracle Cloud Infrastructure and supports the Oracle ACE Apprentice Product Usage milestone.
